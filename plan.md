## שם המוצר
SOS Landing Vault

## מטרות
- הפעלה של ממשק גרירה לשלוש חבילות בסיס: HTML, CSS, JS.
- איחוד התוכן לקובץ HTML יחיד באמצעות הטמעת הסגנונות והסקריפטים פנימה.
- העלאה אוטומטית ל-Blossom לקבלת קישור מיידי.
- מתן סטטוס זמינות של השרת הציבורי בזמן אמת.
- חוויית משתמש רספונסיבית למובייל.

## תכולה
- יצירת תיקייה ייעודית `landing-storage/` עם קבצי HTML, CSS, JS, לוגו וסקריפט.
- בניית עמוד ראשי כולל כותרת, תפריט, אזור גרירה, אזור סטטוס ותוצאות.
- עיצוב כהה מודרני, אנימציית רקע קלה, לוגו SVG מקוריים.
- בדיקת זמינות Blossom באמצעות בקשת HEAD והצגת התוצאה למשתמש.
- ניהול שגיאות ברור: קבצים חסרים, חוסר התאמה, כשל בהעלאה.
- הצגת הקישורים שהתקבלו ותבנית להטמעתם בעמודים עתידיים.

## משימות
1. יצירת לוגו וקובץ CSS מותאם.
2. פיתוח מבנה HTML רספונסיבי עם אזור גרירה.
3. כתיבת `uploader.js` לטיפול גרירה, איחוד קבצים, העלאה והתראות.
4. שילוב `config.js` ו-`blossom.js` להבטחת שימוש במפתח Nostr קיים.
5. הוספת בדיקת זמינות ל-Blossom והצגת זמן תגובה למשתמש.
6. בדיקה בדסקטופ ומובייל, התאמת פונט ועימוד.
7. תיעוד הנחיות שימוש בקובץ זה ובממשק עצמו.

## סיכונים
- חוסר מפתח Nostr יבלום העלאה; נדרש להפנות להתחברות.
- Blossom עלול להגביל HTML; יש לבצע טיפול בשגיאה ולהציע הוראות חלופיות.
- קלט HTML מורכב עלול לכלול קישורים חיצוניים; ההתמעה תתייחס לכך ותחזיר אזהרה במקרה הצורך.

## בדיקות
- העלאת חבילת בדיקה (index.html עם `<link>` ו-`<script>`): וידוא שהייצוא המוטמע עובד.
- הפעלת בדיקת זמינות Blossom: תוצאה מוצגת תוך פחות מ-3 שניות.
- בדיקה בסמארטפון (1024×768 ו-375×667) לאישור רספונסיביות.
